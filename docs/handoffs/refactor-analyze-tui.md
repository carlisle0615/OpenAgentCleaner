# Refactor Analyze TUI Handoff

## 变更说明

本次重构大幅改进了 `Analyze` TUI（终端交互界面）的视觉设计，以提升用户体验并匹配更现代的 SaaS 风格（Modern SaaS Design）。相关更变仅处理了前端渲染展示内容，没有改动 `analyze.go` 和 `analyze_tui.go` 的内部核心处理逻辑。

主要的变更如下：

- **颜色调整（自适应）**：重写了 `analyzeStyles`，由最初的硬编码固定亮色转变为基于 `lipgloss.AdaptiveColor` 的明暗模式自适应系统。现可完美适配 macOS 浅色乃至白底终端，不会再出现“太亮而看不清”及伤眼等问题。
- **界面紧凑化与轻度无边框（Mole 风格）**：
  - 移除了 `lipgloss.RoundedBorder()` 等强行全屏边框以及背景填充。
  - 在左侧列表的右边缘补充了自然的单实线屏障（BorderRight），划分区域以避免完全无边框导致的“列表与详情贴紧无法分辨”。
  - 对过长的 Session Label 执行了截断 `trimForDisplay(str, 22)` 处理，以此消灭了因终端窄宽或长 ID 名字引发的布局彻底崩散（挤出多行的 Bug）。
  - 所有 List 使用 `❯` 作为激活高亮，仅改变前景强调色，消除长短不一的色块。
  - Header 与 Footer 转变为简单统一的暗色提示序列和面包屑，整体对齐在代码或详情上。

## 验证结论

- ✅ **核心可用性**：运行 `make build` 和单元测试 `make test` 均顺利通过，确认不存在语法和类型错误。
- ✅ **视觉稳定性**：TUI 代码的宽度与高度适应性得以保留（在小于 70 宽度时依然可以采用上下排列形式展示）。

## 未解决项与风险追踪

- 本次仅限于 UI/UX 设计升级，并不改变底层存储解析功能。暂未感知风险点。

（_由于 TUI 的交互性质，新设计在长名称路径下的换行裁切表现需在未来的实际使用中持续检验，如遇问题可调整相关的 `lipgloss.Width()` 值。_）
