# Security Policy

## Supported Versions

This project is early-stage. Security fixes will be made on the default branch.

## Reporting a Vulnerability

Please do not open a public issue for deletion-safety bugs, privilege-escalation bugs, or vulnerabilities involving secret exposure.

Instead, report the issue privately to the maintainer through GitHub security reporting if enabled for the repository. If private reporting is not available yet, contact the repository owner directly and include:

- affected version or commit
- operating system version
- reproduction steps
- expected behavior
- actual behavior
- impact assessment

You should receive an acknowledgement within a reasonable time after the report is received.
